Testing the Changes Locally:
============================

1. Build the frontend:
   cd frontend && npm run build
   
2. Verify dist/ directory created:
   ls -la frontend/dist/

3. Start backend:
   uvicorn backend.main:app --host 0.0.0.0 --port 8000

4. Access at: http://localhost:8000

The backend now serves:
- API routes: /api/trades, /api/positions, etc
- WebSocket: /ws/live
- Frontend static files: / (serves index.html + assets)

Key Architecture Points:
========================

Single Service Design:
- One container runs FastAPI backend
- Backend serves both API and React frontend
- No separate frontend server needed
- No external CDN required
- Perfect for Railway/Render/Fly.io

URL Resolution:
Development:
  - Frontend: http://localhost:5173 (Vite dev server)
  - API: http://localhost:8000/api (proxied by Vite)
  - WebSocket: ws://localhost:8000/ws/live

Production:
  - Frontend: https://your-domain.com (served by backend)
  - API: https://your-domain.com/api (same backend)
  - WebSocket: wss://your-domain.com/ws/live (same backend)

Safety & Configuration:
=======================

All platforms default to:
- PAPER_TRADING=true (simulated trades)
- LIVE_TRADING=false (no real trading)
- DEFAULT_RISK_MODE=LOW_RISK (safest settings)
- LOG_LEVEL=INFO (standard logging)

To enable live trading:
1. Set LIVE_TRADING=true in platform dashboard
2. Add MEXC_API_KEY and MEXC_API_SECRET
3. Platform auto-restarts with new config
4. Monitor closely!
